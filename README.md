# Bloom — brand assets

Logo, wordmark, and color guidelines for **Bloom** (`/bloom`, [@bloom_directory](https://x.com/bloom_directory)).

All logo source lives in [`logo/`](logo/), generated by [`logo/build.mjs`](logo/build.mjs). Each mark ships as a vector `.svg` and a 1024×1024 `.png`.

---

## Colors

| Name | Hex | RGB | Use |
|------|-----|-----|-----|
| Cream | `#f0ebe0` | 240, 235, 224 | Primary background; banner and cream icon |
| Burgundy | `#9d2d3f` | 157, 45, 63 | Primary mark color — the filled petals |
| Burgundy deep | `#7a2230` | 122, 34, 48 | Secondary — the outlined petals (the two-tone depth) |
| Burgundy mid | `#8a2a3a` | 138, 42, 58 | Wordmark slash accent; single-color (mono) mark |
| Ink | `#1a1a1a` | 26, 26, 26 | Mono mark on light surfaces |
| Black | `#000000` | 0, 0, 0 | Black icon background |
| White | `#ffffff` | 255, 255, 255 | White icon background |

The mark is two-tone by default: **Burgundy** filled petals alternating with **Burgundy deep** outlined petals. **Burgundy mid** is the tone of the wordmark slash and is used for the single-color version of the mark.

## Wordmark

The `/bloom` wordmark is set in **Instrument Serif** — <https://fonts.google.com/specimen/Instrument+Serif>.

- Style: lowercase, **italic**, as shown on the banner.
- The leading slash `/` is set in **Burgundy mid** (`#8a2a3a`); the word `bloom` is set in **Ink** (`#1a1a1a`) on cream, or reversed to **Cream** on dark.
- Pair the wordmark with the mark only when there's room; at small sizes use the mark alone.

Load the webfont:

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Instrument+Serif:ital@0;1&display=swap" rel="stylesheet">
```

```css
font-family: "Instrument Serif", serif;
font-style: italic;
```

## The mark

A six-petal bloom arranged as a braided pinwheel: filled and outlined petals alternate and weave consistently over/under all the way around, with an outlined petal at the top. The mark is centered with a uniform edge margin (~15–16%, fills ~69% of the frame vertically).

## Logo variants

### Primary (transparent background)

| File | Mark | Use |
|------|------|-----|
| [`bloom-primary`](logo/bloom-primary.svg) | Two-tone burgundy | Default mark. Place on cream, white, or light surfaces. |

### Icons (full square background)

Edge-to-edge square fill — use as app icons, avatars, favicons, or on busy surfaces.

| File | Background | Mark | Use |
|------|-----------|------|-----|
| [`bloom-icon-cream`](logo/bloom-icon-cream.svg) | Cream | Two-tone burgundy | Default icon, brand surfaces |
| [`bloom-icon-white`](logo/bloom-icon-white.svg) | White | Two-tone burgundy | Icon on white UI |
| [`bloom-icon-black`](logo/bloom-icon-black.svg) | Black | Cream | Icon on dark — burgundy lacks contrast on black, so the mark goes light |

### Mono (transparent background, single color)

For one-color contexts (print, embroidery, watermarks) and overlays.

| File | Mark color | Use |
|------|-----------|-----|
| [`bloom-mono-black`](logo/bloom-mono-black.svg) | Ink `#1a1a1a` | One-color on light backgrounds |
| [`bloom-mono-white`](logo/bloom-mono-white.svg) | White | On dark backgrounds / photos |
| [`bloom-mono-cream`](logo/bloom-mono-cream.svg) | Cream `#f0ebe0` | On dark backgrounds / photos (softer than white) |
| [`bloom-mono-burgundy`](logo/bloom-mono-burgundy.svg) | Burgundy mid `#8a2a3a` | Single-color brand mark on light backgrounds |

> The icons carry a small center "knockout" (a dot matching the background) where the petals converge. The transparent and mono marks omit it and let the petals meet, since there's no background color to knock out against.

## Regenerating

Edit the variant list, colors, stroke weight, or margin scale in [`logo/build.mjs`](logo/build.mjs), then:

```bash
cd logo
node build.mjs                                   # write the .svg files
for f in bloom-*.svg; do                          # render 1024px .png files
  rsvg-convert -w 1024 -h 1024 "$f" -o "${f%.svg}.png"
done
```

A single `SCALE` constant in the script controls the edge margin across every asset.
